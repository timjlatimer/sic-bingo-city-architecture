# The Bingo Card Protocol — Universal Interaction Pattern

**Source:** BINGO_CARD_PROTOCOL.md (Pearl's Brain)  
**Date:** March 11, 2026

---

## What the Bingo Card Is

The Bingo Card is the **universal input/output format** for the entire HB1000 system. It is not a gamification gimmick. It is the fundamental interaction pattern through which every piece of the Pearl/HB1000 ecosystem communicates with the user.

Every interaction Pearl has with the user can be expressed as a Bingo Card. The Bingo Card replaces traditional task lists, forms, dashboards, and wizards with a single, universally understood visual pattern.

---

## Why Bingo Works

| Requirement | How Bingo Satisfies It |
|-------------|----------------------|
| **Universally understood** | Bingo exists worldwide across cultures |
| **Visual** | You can see progress at a glance |
| **Finite** | A card has a fixed number of squares — reduces overwhelm |
| **Completable** | You can "win" — provides a dopamine hit |
| **Social** | Bingo is traditionally a community game |
| **Flexible** | Any content can fill a square |
| **Non-hierarchical** | You can complete squares in any order |
| **Low cognitive load** | No learning curve — Ruby Red already knows how bingo works |

---

## Bingo Card Architecture

### Card Level
- Each card = a context (North Star, daily tasks, setup wizard, etc.)
- Each card can have an optional "multi" (AI agent) attached
- Multi = imported Grace that takes care of the work
- Cards can contain graphics, infographics, data
- Cards have KPIs: completion rate, time to complete, outstanding count

### Square Level
- Each square = a task, question, reflection, or action
- Squares can be completed in any order
- Squares can contain text, images, links, or interactive elements

### Universe Level
- The "Bingo Card Universe" = the user's entire life viewed through cards
- Bingo Card sorting, reporting, filtering
- Visual overlay of all cards
- Phase distribution tracking

## The Initiative Dashboard

The Initiative Dashboard is the **primary view** when Pearl opens. It is a visual grid of all active initiatives, each represented as a Bingo Card showing status, health (green/amber/red), the assigned Grace personality, and a mini-instruction input.

Clicking a card opens the full initiative view with its Lean Canvas, task list, communications log, and Learning Loop history.

The Initiative Dashboard replaces the Learning Loop dashboard as the default home screen. The Learning Loop dashboard becomes a secondary tab.

**Bingo City is the visual representation of the Initiative Dashboard** — the buildings ARE the initiative cards, rendered as an isometric city.

---

## Bingo Card KPIs

Every Bingo Card generates metrics that Pearl tracks:

- Outstanding card count
- Phase distribution (how many cards in each life area)
- Completion rates
- Time to complete
- Escalation frequency
- Multi (agent) utilization

---

## Ticker Tape Integration

The Bingo Card system integrates with a running status bar at the bottom of the dashboard:

- Up to two ticker tapes
- Customizable feeds: peer feed, multi-feed, personal feed, world feed
- Shows: outstanding Bingo Cards, PTK counts, other customizable metrics
- Ticker tape setup is itself a Bingo Card (self-referential design)

---

## PTK Engine Integration

Every Bingo Card has a standard PTK (Promises To Keep) engine built in. Each card has a "genie" that watches PTKs within that card's context. PTKs are tracked in a central PTK database and counts are shown in the ticker tape.

---

## The Gamification Engine Connection

The Bingo Card is the **delivery mechanism** for the Gamification Engine. Every card completion, streak, and community contribution feeds into the Gamification Engine, which calculates progress and rewards. The Ticker Tape is the primary output — displaying personalized, non-shaming gamification updates.
