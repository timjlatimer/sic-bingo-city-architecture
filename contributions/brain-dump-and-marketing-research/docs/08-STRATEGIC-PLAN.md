## The Three Workstreams

### Workstream A: Make Pearl Operational (Priority 1)

This is the most important workstream. It transforms Pearl from a concept into a working tool.

**A1. Initiative Dashboard (The New Home Screen)** — Visual grid of all active initiatives. Each card shows name, phase, health, assigned Grace, and mini-instruction input. **Bingo City is the visual implementation of this.**

**A2. Gmail & Calendar Watcher (Phase 1: The Watcher)** — Using MCP connectors, Pearl monitors inbox and calendar. Daily morning briefing, new project identification, attention flags, PTK tracking. Implementable immediately.

**A3. Cloud Butterfly Capture (Flypaper)** — Dedicated input area for paste, dictate, or record. Pearl ingests, deduplicates, extracts, and routes to initiative cards.

### Workstream B: Enrich the Framework (Priority 2)

**B1. Moonshot Canvas** — Dedicated tab for exploring moonshots, defining using 10X framework, tracking progress.

**B2. Move 37 Tab** — Extract from BPJ panels into own dedicated tab.

**B3. Update the Whitepaper (V3)** — Incorporate all new concepts into the Pearl Framework document.

### Workstream C: Documentation & Distribution (Priority 3)

**C1. HB1000 & Cloud Butterflies User Manual** — Comprehensive user's guide.

**C2. Pearl Skill Package** — Package Pearl as distributable Manus Skill + Project template.

**C3. Daily Self-Learning Loop (Scheduled Task)** — Morning self-review, AI industry check, "Manus vs. Everybody Else" audit.
