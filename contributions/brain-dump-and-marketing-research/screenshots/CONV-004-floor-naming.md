# CONV-004: Floor Naming Conventions and the F1-F5 Framework

**Type:** Conversation Reconstruction  
**Context:** How the floor naming convention was established for Bingo City buildings  
**Significance:** Establishes the canonical naming framework for all current and future buildings

---

## The F1-F5 Framework

The floor naming convention follows a consistent five-level framework that mirrors the HB1000's own layered architecture:

| Floor | Name | Function | HB1000 Parallel |
|-------|------|----------|-----------------|
| F1 | Foundation | Core infrastructure, essential capabilities, must-have-first | Layer 1-3 (Mystical, North Star, Core) |
| F2 | Processing / Services | The work layer — where the initiative does its primary job | Layer 4-5 (Local, Engines) |
| F3 | Intelligence | Learning, adaptation, data, community integration | Layer 6-7 (Protection, Communication) |
| F4 | Autonomy | Self-operating capabilities, agent integration, scale | Layer 8-9 (Organizational, Certification) |
| F5 | Transcendence | The moonshot layer — what this initiative looks like at 10X | Layer 10-12 (Funding, Governance, Registry) |

---

## Why This Framework Matters

The F1-F5 framework is not just a naming convention. It is a **build sequence**. You cannot build F2 without F1. You cannot build F3 without F2. The floors are not arbitrary — they represent a logical progression from foundation to transcendence.

This means that when you look at a Bingo City building and see that only F1 and F2 are lit up, you can immediately understand: this initiative has solid foundations and is doing its primary work, but has not yet reached the intelligence, autonomy, or transcendence layers.

---

## The "Not All Buildings Need 5 Floors" Decision

A key decision was that not all buildings need all 5 floors. Buildings with 3 floors (Digger Cafe, PTK Promises, Seba Hub) use F1-F3. Buildings with 4 floors (Wisdom Giants, Effn Duck) use F1-F4. Only Cloud Butterfly has all 5 floors.

This was a deliberate choice to reflect the actual architectural complexity of each initiative. Forcing every initiative into a 5-floor structure would create artificial complexity and false equivalence.

---

## Floor Names as Conversation Starters

The floor names (Foundation, Processing, Intelligence, Autonomy, Transcendence) were chosen to be self-explanatory to a non-technical audience. When Ruby Red Maven looks at a building and sees "F3 — Intelligence," she should intuitively understand that this floor is about the building "getting smarter" — learning from what it's doing.

This is consistent with the HB1000's philosophy of using universally understood language rather than technical jargon.
